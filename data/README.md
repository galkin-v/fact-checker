# Данные чеклистов

`checklists.json` содержит три синтетические записи, предназначенные исключительно для проверки
запуска. Они не являются фрагментом оценочной или обучающей выборки.

Промышленные данные должны соответствовать той же схеме и подключаться в режиме только для чтения
через `CHECKLIST_FILE`. Полный чеклист следует хранить в защищенном хранилище артефактов: не
добавляйте его в этот репозиторий и не включайте в Docker-образ.

В пределах каждого продукта значения `id` и `question` должны быть уникальными. Вопросы следует
считать стабильными ключами конкретной версии чеклиста: модель возвращает вопрос дословно, после
чего сервис определяет его идентификатор по этому файлу.
